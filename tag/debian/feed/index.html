Actualmente se encuentra en ‘developer preview release’.
Instalarlo es bastante sencillo, estas son las instrucciones desde un terminal:

Descarga la ultima versión desde http://labs.adobe.com/downloads/flashplayer10.html:

wget http://download.macromedia.com/pub/labs/flashplayer10/flashplayer10_2_p3_64bit_linux_111710.tar.gz

Extrae la librería libflashplayer.so y copiala en /usr/lib/iceweasel/plugins/ si tienes Debian o en /usr/lib/firefox-addons/plugins/ si tienes Ubuntu

$ tar xzvf flashplayer_square_p2_64bit_linux_092710.tar.gz
$ sudo mv -i libflashplayer.so /usr/lib/iceweasel/plugins/ <dt>Cambia el propietario de la libreria
$ sudo chown root: /usr/lib/iceweasel/plugins/libflashplayer.so

Solo por Debian, deshabilita el plugin swfdec-mozilla cambiando el symlink en /etc/alternatives/

$ cd /etc/alternatives/
$ mv flash-mozilla.so flash-mozilla.so.orig
$ sudo ln -s /usr/lib/iceweasel/plugins/libflashplayer.so flash-mozilla.so

El pasado 6 de agosto los responsables del proyecto debian decidieron congelar la versión 6.0 llamada squeeze. Lo que quiere decir que no se añadiran nuevas funciones si no que de lo que ahora se trata es de ‘pulirla’ revisando bugs i demás…

Éste es el primer paso para que se convierta pronto en la versión estable de ésta popular distribución de Linux.

Como características destacamos:

  • Incluirá KDE 4.4.5, Gnome 2.30.0, LXDE 0.5.0, XFCE 4.6.2, X.org 7.5, OpenOffice.org 3.2.1 y otras muchas aplicaciones
  • Las versiones estables de los servidores comunes como: Apache 2.2.16, PHP 5.3.2, MySQL 5.1.48, PostgreSQL 8.4.4 and Samba 3.4.
  • Compiladores i intérpretes como Python 2.6 and 3.1, Perl 5.10, GHC 6.12 and GCC 4.4.
  • DKMS, un framework para generar modulos del kernel de linux cuyo código fuente no reside en en el árbol del código fuente del kernel.
  • Ejecución ordenada basada en la dependencia de init scripts usando insserv, permitiendo la ejecución paralela para arrancar el sistema en menos tiempo.

I como novedad más destacable, incluirá variantes del kernel de FreeBSD para amd64 i i386, que aún estando en una fase inicial permitirán por primera vez que una distribución linux pueda ejecutarse en un kernel no-linux.
